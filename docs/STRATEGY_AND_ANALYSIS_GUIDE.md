# 股票分析逻辑与策略指南

> 本文档详细说明了项目的分析架构、评分体系和11个交易策略。

---

## 一、整体架构：两层分析体系

该项目采用 **"规则引擎 + AI大模型"双层分析架构**：

| 层级 | 模块 | 文件 | 功能 |
|------|------|------|------|
| 第一层 | StockTrendAnalyzer | `src/stock_analyzer.py` | 基于规则的技术分析，输出0-100评分 |
| 第二层 | GeminiAnalyzer | `src/analyzer.py` | LLM大模型综合分析，输出操作建议 |

---

## 二、第一层：规则引擎评分系统

### 2.1 分析流程

按顺序执行6个分析步骤，最后综合打分：

1. 趋势判断 (`_analyze_trend`)
2. 乖离率计算 (`_calculate_bias`)
3. 量能分析 (`_analyze_volume`)
4. 支撑压力分析 (`_analyze_support_resistance`)
5. MACD分析 (`_analyze_macd`)
6. RSI分析 (`_analyze_rsi`)
7. 综合信号生成 (`_generate_signal`)

### 2.2 六大维度评分体系（满分100）

| 维度 | 最高分 | 评判依据 |
|------|--------|----------|
| **趋势** | 30分 | 均线排列：强势多头=30，多头=26，弱势多头=18，盘整=12，弱势空头=8，空头=4，强势空头=0 |
| **乖离率** | 20分 | 与MA5的偏离度：价格略低于MA5时最高(20分)，超过阈值(默认5%)严格扣分(4分) |
| **量能** | 15分 | 缩量回调=15（最佳），放量上涨=12，正常=10，缩量上涨=6，放量下跌=0（最差） |
| **MACD** | 15分 | 零轴上金叉=15（最强），金叉=12，上穿零轴=10，多头=8，空头=2，死叉/下穿零轴=0 |
| **支撑** | 10分 | MA5支撑有效=5，MA10支撑有效=5 |
| **RSI** | 10分 | 超卖=10（最佳），强势=8，中性=5，弱势=3，超买=0（最差） |

### 2.3 交易信号生成规则

```
score >= 75 且趋势为强势多头或多头 => STRONG_BUY (强烈买入)
score >= 60 且趋势为强势/多头/弱势多头 => BUY (买入)
score >= 45 => HOLD (持有)
score >= 30 => WAIT (观望)
趋势为空头/强势空头 => STRONG_SELL (强烈卖出)
其他 => SELL (卖出)
```

### 2.4 关键技术指标定义

#### 均线排列判断

- MA5 > MA10 > MA20 且间距扩大 → 强势多头 (趋势强度 90)
- MA5 > MA10 > MA20 → 多头排列 (趋势强度 75)
- MA5 > MA10 但 MA10 <= MA20 → 弱势多头 (55)
- MA5 < MA10 < MA20 → 空头排列 (25)
- 均线缠绕 → 盘整 (50)

#### 乖离率

- 公式：`bias = (现价 - MA5) / MA5 * 100%`
- 默认阈值：5.0%
- 强势趋势时放宽至1.5倍阈值

#### 量能状态

- 缩量：当日量/5日均量 <= 0.7
- 放量：当日量/5日均量 >= 1.5

#### MACD（标准12/26/9参数）

- DIF = EMA(12) - EMA(26)
- DEA = EMA(DIF, 9)
- MACD = (DIF - DEA) * 2
- 金叉：前日 DIF-DEA <= 0 且今日 > 0

#### RSI（6/12/24三个周期）

- 超买阈值：70
- 超卖阈值：30
- 以 RSI(12) 为主要判断依据

## 三、第二层：LLM大模型决策

### 3.1 分析流程

1. 将技术面数据（均线、乖离率、量能、MACD、RSI）格式化为Prompt
2. 注入实时行情（量比、换手率、PE/PB）、筹码分布、财报数据
3. 搜索新闻结果注入Prompt
4. 调用LLM生成结构化JSON输出
5. 解析为 `AnalysisResult` 对象

### 3.2 评分标准

| 分数区间 | 操作建议 | 条件 |
|---------|---------|------|
| 80-100 | 强烈买入 | 多个激活技能同时支持、上行空间清晰、风险已排查 |
| 60-79 | 买入 | 主信号偏积极，允许少量待确认项 |
| 40-59 | 观望 | 信号分歧大、风险与机会均衡 |
| 0-39 | 卖出/减仓 | 主要结论转弱、触发止损条件、重大利空 |

### 3.3 decision_type映射

```
强烈买入 / 买入 / 加仓 => buy
持有 / 观望 => hold
减仓 / 卖出 / 强烈卖出 => sell
```

### 3.4 持仓分类建议

LLM输出两套操作建议：
- `position_advice.no_position`：空仓者的操作指引
- `position_advice.has_position`：持仓者的操作指引

## 四、核心交易理念（基线风控规则）

`src/agent/skills/defaults.py` 中定义的7条基线规则：

| 序号 | 规则 | 说明 |
|------|------|------|
| 1 | **严进策略（不追高）** | 乖离率 > 5% 严禁追高，直接判定"观望" |
| 2 | **趋势交易** | 只做 MA5 > MA10 > MA20 的多头排列 |
| 3 | **效率优先** | 关注筹码集中度（90%集中度 < 15% 表示集中） |
| 4 | **买点偏好** | 最佳买点是缩量回踩 MA5，跌破 MA20 观望 |
| 5 | **风险排查** | 减持、业绩预亏、监管处罚等 |
| 6 | **估值关注** | PE 偏高需标注风险 |
| 7 | **强势趋势放宽** | 强势趋势股可适当放宽乖离率要求 |

## 五、交易策略系统

### 5.1 策略分类总览

| 类别 | 策略数量 | 策略名称 |
|------|----------|----------|
| 趋势类 (trend) | 5个 | 默认多头趋势、均线金叉、缩量回踩、放量突破、龙头策略 |
| 反转类 (reversal) | 1个 | 底部放量 |
| 框架类 (framework) | 4个 | 箱体震荡、缠论、情绪周期、波浪理论 |
| 形态类 (pattern) | 1个 | 一阳夹三阴 |

### 5.2 策略路由机制

`src/agent/skills/router.py` 中的 `SkillRouter._detect_regime` 方法根据技术面数据自动判断市场状态：

- MA对齐为bullish且趋势分数 >= 70 → `trending_up`
- MA对齐为bearish且趋势分数 <= 30 → `trending_down`
- 中性或分数在35-65 → `sideways`
- 成交量重且分数在30-70 → `volatile`

## 六、策略详细说明

### 6.1 默认多头趋势 (`bull_trend`) ⭐ 默认激活

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/bull_trend.yaml` |
| 类别 | trend |
| 适用市场 | trending_up |
| 优先级 | 10 |

**核心逻辑**：
- MA5≥MA10≥MA20多头排列
- 优先"回踩不破"而非"高位追涨"
- 放量突破有效阻力时可提高胜率

**评分调整**：
- 多头排列 + 趋势强度良好：`+12`
- 回踩关键均线后企稳：`+8`
- 放量突破关键阻力：`+10`
- 跌破MA20或趋势转弱：`-12`

---

### 6.2 均线金叉 (`ma_golden_cross`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/ma_golden_cross.yaml` |
| 类别 | trend |
| 适用市场 | trending_up |
| 优先级 | 20 |

**信号判定标准**：
1. **金叉检测**：MA5在最近3个交易日内上穿MA10
2. **量能确认**：金叉日成交量应高于5日均量，量比 > 1.2
3. **趋势背景**：盘整后金叉最强，深度下跌中金叉为弱信号
4. **价格位置**：乖离率 < 5%，避免追高

**评分调整**：
- MA5×MA10金叉配合量能：`+10`
- MA10×MA20金叉：`+8`
- MACD零轴上方金叉：额外 `+5`

---

### 6.3 缩量回踩 (`shrink_pullback`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/shrink_pullback.yaml` |
| 类别 | trend |
| 适用市场 | trending_down, sideways |
| 优先级 | 40 |

**入场判定标准**：
1. **前提条件**：股票必须处于上升趋势（MA5 > MA10 > MA20）
2. **回踩检测**：价格回踩至MA5附近（误差1%）或MA10附近（误差2%）
3. **缩量特征**：回调期间成交量 < 5日均量的70%
4. **反弹信号**：当前价格守住均线支撑位，MA5乖离率 < 2%

**评分调整**：
- 缩量回踩MA5：`+10`
- 缩量回踩MA10且量能 < 0.6倍均量：`+8`
- 理想买点设在MA5水平，止损设在MA20

---

### 6.4 放量突破 (`volume_breakout`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/volume_breakout.yaml` |
| 类别 | trend |
| 适用市场 | trending_up |
| 优先级 | 30 |

**突破判定标准**：
1. **阻力位识别**：通常为20日高点或前期震荡平台顶部
2. **量能确认**：当日成交量 > 5日均量的2倍，量比 > 2.0
3. **价格确认**：收盘价必须站上阻力位，收盘应在当日振幅上方30%
4. **乖离率检查**：仍需 < 5%，避免追高

**评分调整**：
- 放量突破确认：`+12`
- 突破伴随板块共振：额外 `+5`
- 止损设在突破位下方3%

---

### 6.5 底部放量 (`bottom_volume`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/bottom_volume.yaml` |
| 类别 | reversal |
| 适用市场 | trending_down |
| 优先级 | 60 |

**反转判定标准**：
1. **持续下跌确认**：股价从20日高点到近期低点跌幅 > 15%
2. **量能异动**：当日成交量 > 5日均量的3倍，量比 > 3.0
3. **价格企稳**：当日K线收阳，价格守住近期低点
4. **确认因素**：筹码分布健康，平均成本接近现价

**评分调整**：
- 底部放量确认：`+8`
- 配合阳线 + 新闻催化：额外 `+5`

**风险提示**：
- 反转信号，风险高于趋势跟踪
- 仓位建议较小（最多2-3成）
- 止损必须严格（设在近期低点下方）

---

### 6.6 箱体震荡 (`box_oscillation`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/box_oscillation.yaml` |
| 类别 | framework |
| 适用市场 | sideways |
| 优先级 | 50 |

**分析步骤**：
1. **箱体识别**：顶部和底部各至少触碰2-3次方可确认
2. **当前位置判断**：
   - 箱底区域（距支撑≤5%）：买入/加仓信号
   - 箱中区域：观望
   - 箱顶区域（距阻力≤5%）：减仓/止盈信号
3. **箱体宽度**：< 5%不建议参与，5-15%标准箱体，> 15%大箱体

**评分调整**：
- 箱底企稳 + 缩量：`+10`
- 箱底放量攻顶：`+12`
- 箱体向上有效突破：`+15`，转趋势策略
- 处于箱顶区域：`-5`（不追高）
- 箱底有效跌破：`-15`，离场

### 6.7 缠论 (`chan_theory`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/chan_theory.yaml` |
| 类别 | framework |
| 适用市场 | volatile |
| 优先级 | 70 |

**核心框架**：分型 → 笔 → 线段 → 中枢 → 趋势

**分析步骤**：
1. **判断价格结构**：识别震荡中枢还是趋势段
2. **背驰判断**：
   - 顶背驰：价格创新高但MACD红柱面积缩小 → 卖出/减仓
   - 底背驰：价格创新低但MACD绿柱面积缩小 → 买入/加仓
3. **买卖点判定**：
   - 一买（最强）：下跌趋势中，最后一个中枢出现底背驰
   - 二买：离开下跌中枢后的第一次回调不破中枢高点
   - 三买：中枢震荡后向上突破

**评分调整**：
- 底背驰 + 一买信号：`+15`
- 二买/三买共振：`+10`
- 顶背驰 / 趋势向下：`-15`

### 6.8 龙头策略 (`dragon_head`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/dragon_head.yaml` |
| 类别 | trend |
| 适用市场 | sector_hot |
| 优先级 | 90 |

**评估标准**：
1. **板块领涨地位**：该股在板块启动周期中率先上涨或涨停
2. **换手率与动能**：龙头股换手率通常 > 5%，量比 > 1.5
3. **相对强度**：上涨日应跑赢板块2%以上
4. **新闻催化**：板块级催化剂（政策、事件、业绩）

**评分调整**：
- 确认为龙头股：`+10`
- 板块正处于主动轮动期：额外 `+5`

**特殊规则**：乖离率可放宽至7%，但超过10%仍需谨慎

### 6.9 情绪周期 (`emotion_cycle`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/emotion_cycle.yaml` |
| 类别 | framework |
| 适用市场 | sector_hot |
| 优先级 | 100 |

**核心哲学**：市场参与者的情绪在"恐慌→悲观→怀疑→希望→乐观→兴奋→贪婪→狂热"之间循环

**情绪阶段量化指标**：
- 换手率 < 0.5%/日：市场冷淡，潜在底部区域
- 换手率 0.5%~2%：正常交投
- 换手率 2%~5%：活跃，不宜追高
- 换手率 > 5%：高热度，警惕情绪顶
- 换手率 > 10%：极度过热，通常为短期顶部

**情绪底部特征（买入区）**：
- 近20日换手率处于近一年低位
- 成交量持续萎缩，低于近60日均量50%以上
- 近期新闻以低调、中性或负面为主
- 股价在MA20附近或以下
- 机构持仓稳定或小幅增加

**情绪顶部特征（减仓区）**：
- 近5日换手率 > 近20日均值的2倍
- 成交量脉冲式放大
- 新闻以利好兑现、散户追捧为主
- 股价偏离MA5超过8%
- MACD出现顶背离

**评分调整**：
- 情绪底部特征满足3项以上：`+14`
- 情绪底部特征满足全部5项：`+20`
- 情绪顶部特征满足3项以上：`-12`
- 情绪顶部特征满足全部5项：`-20`

### 6.10 一阳夹三阴 (`one_yang_three_yin`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/one_yang_three_yin.yaml` |
| 类别 | pattern |
| 适用市场 | 通用 |
| 优先级 | 110 |

**形态定义（最近5个交易日）**：
1. **第1日**：大阳线（实体 > 股价的2%）
2. **第2-4日**：连续三根阴线或小K线
   - 每根K线最低价不跌破第1日开盘价
   - 成交量应逐步萎缩（量比 < 0.8）
   - 三根K线应收在第1日实体范围内
3. **第5日**：又一根阳线，收盘价突破第1日收盘价

**评分调整**：
- 形态成立 + 趋势看多：`+15`
- 形态成立但趋势不明：`+5`
- 理想买点设在第5日收盘价附近，止损设在第1日开盘价下方

### 6.11 波浪理论 (`wave_theory`)

| 属性 | 值 |
|------|-----|
| 文件 | `strategies/wave_theory.yaml` |
| 类别 | framework |
| 适用市场 | volatile |
| 优先级 | 80 |

**核心原则**：市场按照5浪推进+3浪调整的循环结构运行

**推动浪识别**：
- 第1浪：趋势反转的第一波，成交量温和放大
- 第3浪：最强劲的推动浪，通常放大量，MACD强势
- 第5浪：量能往往弱于第3浪，出现顶背离则即将结束

**调整浪识别**：
- A浪：第一次下跌，成交量较大
- B浪：反弹，力度弱于前期涨幅，成交量萎缩
- C浪：第二次下跌，力度往往超过A浪

**黄金位置**：
- 第2浪回调通常在第1浪的38.2%~61.8%
- 第3浪目标通常是第1浪的1.618~2.618倍延伸
- 第4浪不得进入第1浪价格区域

**最优买点**：
- 第2浪回调企稳（黄金坑）：最安全买点
- 第4浪回调企稳：次优
- 第3浪初期突破：放量突破第1浪高点时

**评分调整**：
- 第2浪底部企稳（黄金坑）：`+15`
- 第3浪突破确认：`+12`
- 第5浪末端/顶背离：`-10`
- C浪下跌中：`-12`

## 七、完整决策路径

```
数据采集（OHLCV、实时行情、筹码、财报）
    ↓
技术分析（规则层）→ 趋势状态、0-100技术评分、BuySignal
    ↓
新闻搜索 → 近期新闻舆情
    ↓
策略路由 → 根据市场状态选择1-3个激活技能
    ↓
LLM分析 → 综合所有数据 + 激活技能指令 + 基线风控规则
    ↓
LLM输出 → sentiment_score(0-100)、operation_advice、decision_type
    ↓
完整性校验 → 检查必填字段，缺失则重试或占位补全
    ↓
后处理 → 映射到颜色标签和emoji
```
